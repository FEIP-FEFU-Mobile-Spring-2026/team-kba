# team-kba — Mobile Store

## О проекте
Приложение интернет-магазина одежды. Реализованы экран каталога, детали товара, загрузка данных из API с кэшированием в Room и оффлайн-режим.

## Стек технологий
- **Платформа:** Android
- **Язык:** Kotlin
- **UI:** XML layouts, Material Design 3, Bottom Sheet
- **Загрузка изображений:** Glide
- **Асинхронность:** Coroutines + Flow
- **Архитектура:** MVVM (ViewModel + Repository)
- **Сеть:** Retrofit + OkHttp
- **Кэширование:** Room Database
- **Навигация:** Custom Bottom Navigation

## Реализованный функционал (Блок 4)

### Данные
- [x] Загрузка товаров из API (`GET /catalog`) через Repository
- [x] Авторизация через Bearer token
- [x] Категория «Новинки» формируется на клиенте (товары с тегом `"New"`)
- [x] Категория «Новинки» отображается первой в списке табов
- [x] Поддерживаются поля: размеры (`sizes`), материал, вес, сезон, страна производства

### Каталог
- [x] Переключение табов фильтрует товары по категории
- [x] Карточка товара соответствует макету (отступы, шрифты, цвета, скругления)
- [x] Карточка отображает: изображение, название, цену (формат: `1 234 ₽`)
- [x] Индикатор загрузки (ProgressBar) при старте
- [x] Состояние ошибки с кнопкой повтора

### Детали товара (Bottom Sheet)
- [x] Открывается по тапу на карточку товара
- [x] Отображает: изображение, название, полное описание, цену, теги (`tags`), размеры (`sizes`)
- [x] Кнопка «В корзину» показывает уведомление о добавлении
- [x] Кнопка (i) открывает диалог с характеристиками (материал, вес, сезон, страна)
- [x] Bottom sheet закрывается свайпом вниз или по затемнению

### API и оффлайн
- [x] Данные загружаются из реального API через Repository
- [x] Кэширование в Room Database (не SharedPreferences, не JSON-файл)
- [x] Стратегия **cache-first**: сначала показывается кэш, параллельно идёт запрос к API
- [x] При успешном ответе кэш и UI обновляются
- [x] При отсутствии сети отображаются закэшированные данные
- [x] Snackbar «Нет сети» при отсутствии соединения (если кэш не пуст)
- [x] Если кэш пуст и сети нет — корректное состояние ошибки с кнопкой повтора

### Архитектура
- [x] Данные загружаются через Repository (не напрямую в ViewModel)
- [x] Отдельные слои: data (Room, API), domain (model), presentation (ViewModel, Activity)

### Lifecycle
- [x] Поворот экрана не крашит приложение и не сбрасывает состояние
- [x] Убийство процесса → восстановление состояния (SavedStateHandle + ViewModel)

### Git
- [x] Осмысленные названия коммитов

## Как запустить
1. Клонировать репозиторий
2. Открыть проект в Android Studio
3. Синхронизировать Gradle
4. Запустить на эмуляторе или устройстве (minSdk = 24)
5. При первом запуске потребуется интернет для загрузки данных из API
